# Neural Cryptanalysis — Project Report

## Overview

This project implements a complete neural cryptanalysis framework that trains machine learning models to distinguish reduced-round block ciphers from random permutations. The approach is inspired by Gohr (2019) and the broader field of differential neural cryptanalysis. Given a pair of ciphertexts `(C, C')` produced from plaintexts with a fixed XOR difference `ΔP`, the models learn to predict whether the outputs came from a real cipher or a random permutation.

## Problem Statement

Let `F` be an unknown function — either a reduced-round block cipher `E_k` or a uniformly random permutation `π`.

Given input pairs with a fixed XOR difference:
```
(P, P ⊕ ΔP)  →  (C, C') = (F(P), F(P ⊕ ΔP))
```

**Goal:** Design an ML-based distinguisher that determines whether `F` is a real cipher or a random permutation by learning exploitable differential relations.

**Key insight:** Cipher outputs exhibit structured XOR differences due to the algebraic structure of the round function, while random permutations produce uniform noise. This structure is learnable.

We formulate this as a supervised binary classification problem, where the model is trained on labeled pairs:
- Label 1: Outputs generated by a real cipher Ek
- Label 0: Outputs generated by a random permutation π

The model learns a function f(C, C') → {0,1} that predicts whether the input pair originates from a cipher or a random permutation.

**Evaluation Metric**
Model performance is evaluated using classification accuracy on a held-out test set. A baseline accuracy of 50% corresponds to random guessing. Any accuracy significantly above 50% indicates that the model is able to distinguish cipher outputs from random permutations.

**To analyze the effectiveness of neural distinguishers, we systematically evaluate performance across multiple dimensions:**
- Number of rounds (to study the effect of diffusion and security)
- Cipher type (to compare structural differences across designs)
- Input representation (to understand how feature encoding impacts learning)
- Model architecture (to evaluate different learning capabilities)

A successful distinguisher achieves accuracy significantly above 50%, which corresponds to random guessing. As the number of rounds increases, the cipher is expected to become indistinguishable from a random permutation, and the model’s accuracy should approach 50%.

Different input representations of (C, C') are explored, including raw concatenation, XOR differences (delta), and statistical feature encodings. These representations expose different aspects of the underlying structure, influencing the model’s ability to detect non-random patterns.

The maximum number of rounds for which the model achieves accuracy above random guessing provides an empirical estimate of the cipher’s resistance to distinguishability. This serves as an experimental measure of the cipher’s security margin.

## Dataset Generation

Datasets are generated as balanced binary classification problems:

1. Sample random plaintext `P` using `os.urandom`
2. Compute paired plaintext `P' = P ⊕ ΔP` (fixed Gohr-style difference)
3. Encrypt both: `(C, C') = (E_k(P), E_k(P'))`
4. Label: `y = 1` (cipher) or `y = 0` (random permutation)
5. Apply chosen representation transform to `(C, C')`

**Fixed input differences (ΔP) per cipher:**

| Cipher | ΔP |
|--------|-----|
| simon32 | `0x40000000` |
| gift64/skinny64/craft/present | `0x0000000000000001` |
| gift128/skinny128 | `0x...0001` (128-bit) |
| pyjamask | `0x...0001` (96-bit) |

## Machine Learning Models (4 total)

# Training Details

All models were trained using a supervised binary classification setup. The dataset consisted of labeled pairs indicating whether the outputs were generated by a real cipher or a random permutation.

* **Optimizer:** AdamW
* **Loss Function:** Binary Cross-Entropy (BCE)
* **Batch Size:** 128
* **Epochs:** 6 (reduced for efficiency while maintaining convergence)
* **Learning Rate:** 1e-3 (with minor tuning for specific models)
* **Train/Test Split:** 80% training, 20% testing
* **Data Shuffling:** Applied before splitting to avoid ordering bias

For certain models such as MINE, a slightly larger dataset and more epochs were used to ensure stable convergence.

All experiments were conducted using the same training setup unless otherwise specified, ensuring consistency across comparisons.

---

# Evaluation Metric

Model performance is evaluated using **classification accuracy** on a held-out test set.

* A baseline accuracy of **50% corresponds to random guessing**, since the task is binary classification.
* Any accuracy significantly above 50% indicates that the model is successfully distinguishing between cipher-generated and random outputs.

As the number of rounds increases, a secure cipher is expected to behave like a random permutation. Therefore, the model’s accuracy should approach 50%.

The point at which accuracy drops close to this baseline is used as an indicator of the cipher’s resistance to distinguishability.

---



### 1. MLP — Multi-Layer Perceptron
- **Architecture:** `Linear(1024) → LN → GELU × 5 + 2 skip connections → Linear(1) → Sigmoid`
- **Params:** 753,537 (delta rep) / 804,737 (concat rep)
- **Best rep:** delta
- **Best accuracy:** 1.0000 (r=4, delta, N=4000, 15 epochs)

### 2. CNN — Convolutional Neural Network
- **Architecture:** 3-channel input (C, C', XOR) → MultiScale stem (k=1,3,5) → 3 ResStages → global avg+max+std pooling → MLP head
- **Params:** 313,141
- **Best rep:** concat / bitslice
- **Best accuracy:** 0.9600 (r=4, concat, N=4000, 15 epochs)

### 3. Siamese Network
- **Architecture:** Shared encoder (2-layer) → concat embeddings + |diff| → classifier
- **Params:** 24,897
- **Best rep:** concat
- **Best accuracy:** 0.9975 (r=4, concat, N=4000, 15 epochs)

### 4. MINE — Mutual Information Neural Estimator
- **Architecture:** Augmented input `[C ‖ C' ‖ ΔC]` → `Linear(512) → LN → GELU × 4 + 2 skip connections → Linear(1) → Sigmoid`
- **Params:** 540,929
- **Best rep:** concat (augmented with XOR channel internally)
- **Best accuracy:** 0.9938 (r=4, concat, N=4000, 15 epochs)
- **Reference:** Belghazi et al., ICML 2018

# Experimental Protocol

To ensure fair and meaningful comparisons, experiments were designed by varying only one factor at a time while keeping all other parameters fixed.

* When comparing **models**, the representation and dataset were kept constant
* When comparing **representations**, the model and training setup were fixed
* When comparing **ciphers**, the same model and representation were used

This controlled setup isolates the effect of each component and ensures that observed differences in performance are attributable to the factor under study.

## Key Findings

1. **Statistical representation** achieves the highest accuracy (99.83%) — the 59-feature vector captures differential structure better than raw bits alone.
2. **Delta representation** is the most direct — encodes `C ⊕ C'` bits, achieving 99.58% at r=4 for simon32.
3. **MLP** achieves 100% accuracy at r=4 with delta rep — the simplest model wins with the right representation.
4. **Siamese network** is the most parameter-efficient model (25K params) with 99.75% accuracy.
5. **MINE** with XOR augmentation achieves 99.38% at r=4 — the explicit differential channel is critical.
6. **Cipher hardness** varies significantly: CRAFT becomes indistinguishable at r=4, while SIMON32 remains distinguishable up to r=8.
7. **pyjamask, skinny64, skinny128** are perfectly distinguishable (100%) at their best rounds.
8. All 8 ciphers pass test vector verification and avalanche effect checks.

# Limitations and Future Work

While the experiments provide strong evidence of the effectiveness of neural distinguishers, several limitations remain:

* The study focuses on **reduced-round ciphers**, and results may not directly extend to full-round secure implementations
* The analysis relies on **fixed input differences**, and does not explore optimal differential selection
* Models are trained on synthetic datasets, which may not fully capture real-world attack constraints
* Computational limitations restrict the scale of experiments and model complexity

## Future Work

* Automated search for effective input differences
* Comparison with classical differential cryptanalysis methods
* Extending the approach to partial key-recovery attacks
* Exploring transfer learning across different ciphers or round configurations

---

# Conclusion

In this work, we implemented and evaluated machine learning-based distinguishers for reduced-round block ciphers. The results demonstrate that neural models can successfully learn structured relations between plaintexts and ciphertexts without explicit cryptanalytic rules.

Among the factors studied, **input representation plays the most critical role**, with delta and statistical representations consistently achieving the highest performance. Model architecture also influences results, with Siamese and CNN models performing well in capturing relational patterns, though their effectiveness depends on compatibility with the input representation.

Across all ciphers, accuracy decreases as the number of rounds increases, eventually approaching random guessing. This behavior reflects the increasing diffusion and security of the cipher, and allows us to empirically estimate the maximum number of distinguishable rounds.

Overall, the study highlights the potential of machine learning as a tool for cryptanalysis, while also emphasizing that the effectiveness of such methods is fundamentally constrained by the design and security properties of the underlying cipher.
